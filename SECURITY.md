# 🔐 Security Policy

## 🛡️ Supported Versions

Only the latest stable versions receive security updates.

| Version | Supported |
|---------|-----------|
| **1.x.x** | ✅ Fully Supported |
| **0.x.x** | ❌ Not Supported |
| **Pre-release / Custom forks** | ❌ Not Supported |

---

## 🐞 Reporting a Vulnerability

We take security seriously.  
If you discover a vulnerability, **do NOT** open a public GitHub issue.

Instead, please report it privately:

### 🔒 How to Report

Email: **advrter@gmail.com**  
Subject: **[SECURITY] Vulnerability Report**

Include the following:

- Description of the vulnerability  
- Steps to reproduce  
- Impact or affected components  
- Proof of concept (if available)  
- Suggested fix (optional)  

---

## 🔁 Response Time Expectations

- **Acknowledgment:** within **48–72 hours**  
- **Investigation:** within **7–14 days**  
- **Fix:** as soon as possible after confirmation  
- You will be notified when:
  - Issue is verified  
  - Patch is released  
  - Public advisory (if any) is planned  

---

## 🔧 Responsible Disclosure Rules

Please follow these guidelines:

- Do **not** publicly disclose the issue before a fix is available  
- Do **not** exploit the vulnerability  
- Avoid actions that could disrupt services or cause data loss  

---

## 📌 Security Recommendations for Users

To stay safe:

- Always update to the latest version  
- Keep API keys and credentials secure  
- Never commit secrets to the repository  
- Use limited-permission tokens when possible  
- Monitor logs for suspicious activity  

---

## 🙏 Appreciation

Thank you for helping keep this project secure.  
Your responsible disclosure protects the entire community.
